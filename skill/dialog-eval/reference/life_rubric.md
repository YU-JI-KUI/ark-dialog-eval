## 解决度判据(可观察标准,不依赖深度寿险知识)

> 评测者不懂寿险条款没关系——按下表看"答案有没有给到用户要的那类东西""用户下一轮满不满意"即可。

| 意图 | 算「解决」(yes) | 算「未解决」(no/partial) |
|------|------------------|--------------------------|
| 保单查询 | 给出了保单的具体状态/数值(保额、生效日等),用户未重问 | 只说"请登录APP查"未给信息、用户重问 |
| 理赔咨询 | 给出理赔步骤 + 所需材料清单 | 含糊带过、用户追问"具体要哪些材料""怎么报" |
| 缴费续期 | 给出缴费时间/金额/方式中用户问的那项 | 答非所问、用户重问缴费细节 |
| 保全变更 | 给出变更的具体办理路径/入口/所需材料 | 只说"支持变更"不给办理方式 |
| 投保咨询 | 针对用户情况(如"高血压能买吗")给出明确可否/条件 | 回避、只给通用话术、用户追问 |
| 产品咨询 | 答到用户问的责任/条款/对比点 | 答非所问、堆砌无关产品介绍 |
| 万能账户 | 给出账户价值/利率的具体数值或查询路径 | 只给概念不给数值/路径 |
| 贷款咨询 | 给出额度/利率/申请方式中用户问的那项 | 含糊、用户重问 |
| 咨询客服 | 给出转人工入口或客服电话 | 未给联系方式 |
| 拒识 | —(非寿险业务,answer_resolved 通常 no) | 被错误地用寿险答案回应 = 分发错 |
| 活动 | 给出权益内容 + 领取/使用入口 | 只列权益不给入口 |

**寿险特别提示(可信度关键)**:
- 理赔、投保、保全常涉及**金额、条件、材料**等业务事实。这些事实是否正确 **AI 一律不判**(铁律三),标 `factual_verifiable=false`。只判"有没有给到这类信息、用户满不满意"。
- 涉及健康告知、核保结论、理赔金额等**敏感/合规**内容,`compliance_flag=true` 或 `needs_human_review=true`,转人工。

## 分发判据

- 系统分发到的意图/模块与问题应去向一致 → dispatch_correct=true。
- 拒识类(查股票、手机充值等非寿险业务)被分到任何寿险意图 → false。
- 承接模块参考:保单助手(保单查询/保全/缴费)、理赔助手(理赔)、智能顾问(投保/产品/万能/贷款)、客服(咨询客服/活动)。
