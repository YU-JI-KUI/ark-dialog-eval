# 对话评测报告模板

> 生成报告时严格按本结构与口径。目的是让**不同 BU、不同批次的报告可横向对比**,
> 也让不懂业务的人能读懂"结果可信吗、哪里要改"。所有数字必须来自脚本计算,不要估算。

| 指标 | 数值 |

**如何提升 BU 分发准确率**:{结合两类错误占比给建议——漏多→证券业务分类覆盖不全/分发阈值过严;误收多→拒识规则缺失,他业务问题被错收。举具体例子。}

## 2. 可信度结论(校准模式必填,生产模式跳过)

> 这一节回答:**"这套自动评测可信吗?"** —— 用 AI 判断与人工金标的一致性说话。

| 评测维度 | 一致率 κ | 判定 | 准确率 |
|----------|----------|------|--------|
| 分发是否正确 | {κ} | {高度一致/可信/需改进/偏低} | {x}% |
| 答案是否解决 | {κ} | {…} | {x}% |

**结论一句话**:{例如"答案解决度判断 κ=0.71,达到可信阈值,可逐步替代人工;分发判断 κ=0.55 偏低,见下方不一致 case 分析。"}

> κ ≥ 0.8 高度一致 / ≥ 0.6 可信 / 0.4–0.6 需改进评判规则 / < 0.4 标准或人工标注需对齐。

## 6. 方法论声明(固定附在报告末尾,建立信任)

本评测遵循四条铁律:① 每个判断附可观察依据;② 解决度只看相关性/完整性/用户下一轮行为,不靠业务对错;③ 事实正确性一律标"不可验证",不假装懂业务;④ 不确定即转人工。事实类判断未纳入自动评测,需人工复核的部分见第 1 节计数。
