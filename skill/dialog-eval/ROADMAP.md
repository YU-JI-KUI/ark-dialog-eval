# 从 0 到 1 落地路线图 —— 给"不知道从哪开始"的你

> 你的处境:输入只有一份日志 Excel;不同人要不同指标;你不懂业务细节;业务规则要不全。
> 这份文档把这团乱麻拆成**有先后顺序的台阶**,你每次只用爬一级。
> 核心原则:**先有一个能跑通的、糙的、端到端的结果,再逐步加细。绝不一开始就追求完整。**

---

## 0. 先记住三句话(你焦虑的解药)

1. **你不是业务判官,你是规则的搬运工和验证者。** AI(Qwen)做业务判断,你负责给它立规矩、用人工标注验它准不准。不懂业务≠做不了。
2. **4 个评测对象来自同一份 Excel 的不同列,用同一条流水线。** 不是 12 个活,是 1 条流水线 + 几种切法。
3. **不要等规则齐了再开始。** 先用通用规则跑出糙结果,拿结果去问业务"这条判得对吗"——这比凭空问"你的规则是什么"高效十倍。

---

## 1. 把需求翻译成一张表(消除"千头万绪")

你听到的所有需求,都落在这张表里。打印出来贴墙上:

| 评测对象(评什么) | 来自 Excel 哪列 | 谁关注 | 难度 |
|---|---|---|---|
| ① 跨 BU 分发准不准 | 分发BU / 实际应分BU | 分发团队 | 中(注:当前数据 BU 恒证券,无区分度,见 §6 缺口) |
| ② BU 内意图分类准不准 | 模型意图 / 智能体分类 + 金标 | 意图/路由团队 | 中 |
| ③ 答案有没有解决用户 | 客户问题 + 答案 + 下一轮 | 业务/运营 | 中(有通用判据) |
| ④ 多轮里 agent 为何退出/中断 | 会话ID + 轮次 + 智能体名称变化 | 多轮/对话团队 | 高(放第三期) |

| 视角(怎么看) | 实现方式 |
|---|---|
| 切片(不同人看不同模块) | 同一批结果,按维度分组统计(分发的人看①,意图的人看②) |
| 趋势(每天/每周变化) | 单日快照存档 → 多天对比(**第二期**,先有单日) |
| 归因(不对怎么提升) | 每个判断带"依据"+ 失败 case 聚类 → 优化建议 |

**看懂这张表你就赢了一半**:它告诉你"评测对象"和"视角"是正交的,不会爆炸成无数个需求。

## 2. 落地阶段(每一级都能独立交付、独立给业务看)

### 🟢 第 0 级:让一份 Excel 走通,出最糙的报告(目标:1-2 天)
**只做一件事:Excel 进 → 一份 Markdown 报告出。** 哪怕指标很糙。
- 用 `scripts/prepare.py` 把 Excel 解析成干净样本(已写好,直接用)。
- 让 AI 按 SKILL.md 的四条铁律,对**每条**对话判:意图、分发对不对、解决没解决,**每个判断带依据**。
- 按 `reference/report_template.md` 出报告。
- **验收**:能对你手上任意一份 Excel 出一份结构完整的报告。指标准不准先不管。
- **为什么先做这个**:打通"端到端"比"做准某一项"重要 10 倍。有了它,后面全是往里填细节。

### 🟢 第 1 级:用人工金标校准,拿到第一个 κ(目标:+1 天)
**这是你建立信任的关键一步。**
- 用你那 2000 行**有人工标注**的数据跑第 0 级。
- 用 `scripts/metrics.py` 算 AI 判断 vs 人工金标的一致率 κ。
- **验收**:报告里出现"分发 κ=0.x、解决 κ=0.x"。
- **为什么关键**:κ 是你向业务证明"AI 可信"的**唯一客观语言**。在这之前,任何人说"AI 判得不准"你都无法反驳;有了 κ,讨论从"我觉得"变成"数字说"。

### 🟡 第 2 级:看不一致 case,倒逼业务补规则(目标:持续,每周迭代)
**这是解决"我不懂业务/规则不全"的核心机制。**
- 导出 AI 与人工不一致的 case(metrics.py 已输出 `disagreements`)。
- 拿着这些 case **逐条**去问对应业务方:"AI 判解决、你判没解决,凭什么?"
- 业务方的回答 = 一条新规则 → 写进 `reference/<bu>_rubric.md`。
- **验收**:每周 rubric 都长出几条新判据,κ 逐周上升。
- **为什么这样问**:业务方对着具体 case 能说清楚,对着"你的规则是什么"这种空问题说不清楚。**用 case 当钩子,把他们脑子里的隐性规则钓出来。**

### 🟡 第 3 级:做"切片视角",一份报告满足不同人(目标:+2 天)
- 同一批结果,按维度分组聚合:分发团队看①的切片、意图团队看②的切片。
- 每个切片:样本量、准确率/解决率、失败 case Top、优化建议。
- **验收**:分发的人、意图的人、业务的人,各自能在报告里找到自己关注的那一页。

### 🔵 第 4 级:单日快照存档 → 趋势(第二期)
- 每天的报告结果存一份(JSON)。
- 攒够几天 → 对比:哪个意图解决率在涨、哪个模块在退步。
- **验收**:出"本周 vs 上周"对比。**注意:这一级必须先有第 0-1 级稳定产出,否则趋势是在对比噪声。**

### 🔵 第 5 级:多轮 agent 退出分析(第三期,最难,最后做)
- 按会话ID聚合,看一通对话里"智能体名称"怎么变的。
- agent 中途换了/退出了 → 为什么?(用户换话题?当前 agent 接不住?分发又错了?)
- **为什么放最后**:它依赖前面所有维度都判准了,且业务规则最复杂。前面没扎实就做这个 = 空中楼阁。

## 4. 怎么跟业务方开口(解决"我不懂业务"的话术)

不要问:"你们判断答案好坏的规则是什么?"(他们答不上来)

要问(拿着具体 case):
- "这通对话,AI 说解决了,你觉得呢?如果没解决,是答非所问、没说完整、还是用户后面又问了?"
- "这条你说分错了,那它**本应**分到哪个意图?"(这一问能帮你补上最缺的"正确意图"真值)
- "这类问题,什么样的回答你才算它'解决'?给我一个'好'的例子和一个'差'的例子。"

**每次对话,你的产出是 rubric 里多出来的几行字。** 几周后,rubric 就是这个 BU 的"评测宪法",而它是业务方一句句喂出来的——到那时,结果自然被认可,因为标准是他们定的。

---

## 5. 判断你"做对了"的信号

- ✅ 你能对任意一份 Excel 出报告(哪怕糙)→ 第 0 级达标,你已经"开始"了。
- ✅ 报告里有 κ 数字 → 你有了和业务对话的硬通货。
- ✅ rubric 每周变长、κ 每周上升 → 飞轮转起来了,这是最健康的状态。
- ✅ 业务方开始说"这个判得对/不对,应该是…" → 他们入局了,认可只是时间问题。

---

## 6. 已知缺口(诚实告诉业务方,不要假装能做)

- **跨 BU 分发**:当前数据 `BU`/`分发BU`/`实际应分BU` 恒为"证券",**没有区分度**。真正的分发是证券 BU 内的意图级路由(②)。要做真正的跨 BU,需要业务方提供含多 BU 的数据或正确 BU 真值。
- **分错时"正确意图"无干净真值**:目前没有一列记录"分错时本应分到哪"。需要业务方在复核不一致 case 时顺手补(见 §4 第二问)。
- **事实正确性**:无知识库,AI 不判。涉及金额/条件/条款对错的,一律转人工(铁律三)。
- **多轮退出归因**:最复杂,第三期再做,且依赖前面维度判准。

> 把这些缺口明明白白写给业务方。"我现在能做①②③的哪些部分,④要等条件"——诚实的边界比假装全能更让人信任。
